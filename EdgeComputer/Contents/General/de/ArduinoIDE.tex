\chapter{Arduino IDE}

\section{Einleitung}

Zur Programmierung der Ardiono-Boards stellt die Firma Arduino eine Entwicklungsumgebung, die Arduino IDE, zur Verfügung. Die Arduino IDE\index{Arduino IDE} ist eine offizielle Open-Source-Software für Arduino, die zum Bearbeiten, Hochladen und Kompilieren von Codes in das Arduino-Modul verwendet wird. Es handelt sich um eine plattformübergreifende Software, die für Betriebssysteme wie Windows, Linux und macOS verfügbar ist. Sie läuft auf der Java-Plattform und unterstützt eine Reihe von Arduino-Modulen. Sie unterstützt die Sprachen C und C++. Der Programm-Code wird in der Arduino IDE erstellt, dann wird eine Hex-Datei erzeugt und  auf das angeschlossene Arduino-Boards geladen. Das Programm wird als Sketch\index{Sketch} bezeichnet. 
\cite{Arduino:2022},  \cite{ArduinoIDE:2022} 

In diesem Kapitel wird beschrieben, wie die Arduino IDE installiert und verwendet wird. Dabei erfolgt eine Beschreibung der wichtigsten Funktionen und ein Beispiel wird vorgestellt.

\section{Laden eines Programms}


Nach der Konfiguration der Entwicklungsumgebung kann nun das erste Programm auf das angeschlossene   Arduino-Board geladne werden. Der Sketch der im Editor geschrieben wurde, sollte zunächst gespeichert werden. Die Endung eines Sketches ist \FILE{.ino}\index{Sketch!ino}. Sobald die Schaltfläche \glqq Hochladen\grqq{} werden neuere Arduino-Boards automatisch zurückgesetzt. Ältere Arduino-Boards müssen gegebenenfalls durch die Betätigung des Tasters \glqq Reset\grqq{} manuell zurückgesetzt werden. Das Hochladen kann über die LEDs TX iund RX verfolgt werden, da diese während des Vorgangs blinken.

\section{Struktur eines Sketches}

Nach Abschluss der Konfiguration der Entwicklungsumgebung kann man mit der Programmierung eines Sketches gestartet werden. Alle Sketche folgen dem gleichen Ablauf. Zunächst wird eine Initialisierung durchgeführt und anschließend eine weitere Funktion in einer Endlosschleife abgearbeitet.
Daraus folgt, wie auch schon in der Abbildung \ref{ArduinoIDESketch}  ersichtlich ist, dass der grundlegende Arduino-Sketch\index{Arduino!Aufbau} aus zwei Teilen besteht. Der erste Teil ist eine Funktion \PYTHON{void setup()}, die void zurückgibt, und wir machen die Initialisierung wie die Ausgabe-LED-Farbe oder die Angabe des Kerns. Der zweite Teil ist die Funktion \PYTHON{void loop()}, in der wir Funktionen definieren, die während der Schleife ausgeführt werden sollen. Diese Codes werden zwischen Klammern \{ \} platziert und jede Funktion hat einen Rückgabetyp, hier ist es der Rückgabetyp void. Im Folgenden wird dies detailliert erläutert.

\bigskip

Vor der Funktion \PYTHON{setup()} werden Deklarationen von Variablen und Instanzen von Klassen angelegt. 

Die Variablen im Deklarationsbereich sind global. Notwendig sind globale Variablen nicht, aber um eine bessere Dokumentation zu gewährleisten, ist die oft sinnvoll. Zum Beispiel die Festlegung von Ausgängen von LEDs können hier vorgenommen werden, so dass sofort ersichtlich ist, welche Ausgänge belegt sind. Ebenso können hier den Variablen Werte zur Initialisierung zu gewiesen werden.

Um eine Klasse nutzen zu können, muss eine Instanz angelegt werden. Im Deklarationsbereich werden den Instanzen Namen zugewiesen.




Setup()
Jeder Sketch besitzt die Funktion \PYTHON{Setup()}, die beim Start des Arduino-Boards genau einmal aufgerufen wird. Dementsprechend wird hier der Anfangszustand festgelegt. Folgende Punkte können definiert werden:

\begin{enumerate}
  \item Pin-Funktionalität mit Hilfe der pinMode-Funktion \PYTHON{pinMode}
  \item Anfangszustand der Pins
  \item Initialisierung von Klassen
  \item Initialisierung der Variablen
  \item Kodierung der Logik 
\end{enumerate}

 Nach Beendigung der Funktion \PYTHON{Setup()} wird die Funktkion \PYTHON{Loop()} aufgerufen. Der Name drückt schon aus, dass diese Funktion in einer Endlosschleife immer wieder aufgerufen wird. Sie enthält die Logik der Nutzung des Arduino-Boards.
