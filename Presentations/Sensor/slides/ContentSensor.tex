\STANDARD{Allgemeines}
{ 
	
  \textbf{Grundsätzlich:} Sensor (lat: \glqq Fühler" ) dienen zur \textbf{qualitativen} und \textbf{quantitaiven Auffassung} von physikalischen, chemischen, biologischen, klimatischen und medizinischen Größen. \cite{Hering:2023} \\ 
  Jedes Sensorsystem besteht aus folgenden Komponenten: 
  \begin{itemize}
  	\item \textbf{Sensor-Element} z.B. veränderlicher Widerstand
  	\item \textbf{Auswerte-Elektronik} z.B. Mikrokontroller
  \end{itemize}
  Quellen: 
  \cite{Loeffler-Mang:2012}, \cite{Roddeck:2023}
}

\STANDARD{Optische Näherungssensoren}
{ 
Optische Näherungssensoren nutzen Sensorelemente, welche sowohl aus \textbf{Transmitter} als auch aus \textbf{Receiver} bestehen. Hierbei sind \textbf{Strahlenquellen} beispielsweise der \textbf{Sender} und \textbf{Fotodioden} die \textbf{Empfänger}. 
\begin{itemize}
	
	\item \textbf{Einweg-Lichtschranken:} Sender und Empfänger gegenüberliegend. Hohe Auflösung, bis zu 10m \cite{Hering:2023}
	\item \textbf{Reflex-Lichtschranken:} Sender und Empfänger in einem Gehäuse, \textbf{Reflektor}. Einfache Einstellung, bis zu 5m Abstand \cite{Hering:2023}
\end{itemize}
}

\section{Bibliothek}
\STANDARD{Beschreibung der Bibliothek Bounce2}{
\subsection{Description}
Die Bounce2-Bibliothek dient zur \textbf{Entprellung (Debouncing)} von digitalen Eingängen, insbesondere von mechanischen Tastern und Schaltern.

\textbf{Problemstellung}
Mechanische Schalter erzeugen beim Betätigen \textbf{Prellsignale}, d.\,h. mehrere schnelle Zustandswechsel, die als mehrere Eingaben interpretiert werden können.

\textbf{Lösung durch Bounce2}
Bounce2 filtert diese Prellsignale und stellt sicher, dass:
\begin{itemize}
	\item nur stabile Zustandsänderungen erkannt werden
	\item Fehlauslösungen vermieden werden
\end{itemize}
 }

\subsection{Functions}
\STANDARD{Funktionen der Bibliothek}{
	
\begin{itemize}
	\item Überwachung des Eingangszustands über die Zeit
	\item Verwendung eines \textbf{Zeitintervalls (Debounce-Delay)}
	\item Zustandsänderung wird erst akzeptiert, wenn sie stabil bleibt
\end{itemize} 
}
\STANDARD{Wichtigsten Methoden}{
\begin{itemize}
	\item \texttt{attach(pin)}: Verknüpft den Taster mit einem Pin
	\item \texttt{interval(ms)}: Setzt die Entprellzeit
	\item \texttt{update()}: Aktualisiert den Zustand (muss regelmäßig aufgerufen werden)
	\item \texttt{read()}: Gibt den aktuellen stabilen Zustand zurück
	\item \texttt{fell()} / \texttt{rose()}: Erkennen von Flanken (Taste gedrückt/losgelassen)
\end{itemize}
}

\textbf{Hauptfunktionen}\\
Der E18-D80NK fungiert als optische \textbf{Lichtschranke} zur berührungslosen Objekterkennung. Die Hauptmerkmale sind die digitale Signalausgabe, die Unempfindlichkeit gegenüber sichtbarem Licht und die kompakte Bauweise für die Integration in Steuerungssysteme mit einem \textbf{Arduino}.

\medskip

\begin{enumerate}

		\item \textbf{Blaue Ader}: Anschluss an GND (\textbf{Arduino}).
		\item \textbf{Schwarze Ader (Signal)}: Anschluss an Digital Pin 2 (\textbf{Arduino}).
	\end{itemize}
	\item \textbf{Anzeige-Einheit}:
	\begin{itemize}
		\item \textbf{LED} (Anode): Über Vorwiderstand (220 Ohm) an Digital Pin 13.
		\item \textbf{LED} (Kathode): An GND (\textbf{Arduino}).
	\end{itemize}
\end{enumerate}

\textbf{Sicherheitshinweise}\\
\begin{itemize}
	\item Betrieb nur mit stabilen 5V DC.
	\item Signalausgang nicht für hohe induktive Lasten nutzen.
	\item Optik sauber halten, um Fehlmessungen der \textbf{Lichtschranke} zu vermeiden.

\textbf{Fehlerbehebung}\\
\begin{itemize}
	\item \textbf{Dauersignal}: Reichweite am Potentiometer reduzieren.
	\item \textbf{Kein Signal}: Pin-Zuweisung in der \textbf{Arduino IDE} prüfen.
	\item \textbf{Instabilität}: GND-Verbindung zum \textbf{Arduino} sicherstellen.
\end{itemize}

\STANDARD{Kalibrierung}{
\section{Calibration}
Um eine Lichtschranke zu kalibireren ist folgendes vorgehen Empfehlenswert 
\begin{enumerate}
	\item Den Sensor gemäß der Herstellerangaben an einen Mikrokontroller anschließen
	\item Die Lichtschranke montieren, Der Abstand zwischen Sender udn Empfägner bzw. Sender und Reflektor sollte kleiner als der maximale Abstand gewählt werden.
	\item Die Lichtschranke ausrichten, sodass Sender \& Empfänger koaxial ausgerichtet sind. \\ 
	Bei Reflex-Lichtschranken sollten Sender und Reflektor koaxial liegen. 
	\item Ein Skript zum Schalten eines Aktors oder ausführen eines Befehls öffnen und auf den Mikrokontroller spielen. 
	\item Prüfen ob das gelesenen Signal mit Realität einstimmt. \\
	\textbf{in unserem Fall: NC also HIGH-Signal wenn kein unterbrocherner Lichtstrahl}
	\item Bei falschem Signal: Neuausrichtung.
\end{enumerate}
}

\STANDARD{Test der Lichtschranke - Beschreibung}{
\subsection{einfacher Test } 
Auf dem Arduino soll die eingebaute LED dauerhaft leuchten. Die Lichtschranke wird ebenfalls ordnungsgemäß angeschlossen. Nun soll die ordnungsgemäße Funktion der Lichtschranke geprüft werden. Folgendes Vorgehen wird nachgehend beschrieben. 
\begin{enumerate}
	\item Starten der ArduinoIDE
	\item Verbinden der Lichtschranke an die Versorgungspins (5V, GND) des Mikrokontrollers  
	\item Verbinden der Signalleitung der Lichtschranke an einen GPIO Pin des Mikrokontrollers 
	\item Verbinden des Mikrokontrolles mit dem Rechner per USB 
	\item Öfnnen der ArduineIDE
	\item Öffnen des Sketches zur Erprobung
	\item Eintragen des mit der Signalleitung verbundenen GPIO Pins an der vorgesehenn Stelle 
	\item Sketch auf den Mikrokontroller spielen
\end{enumerate}
}


\STANDARD{Beispielcode}{
	\section{Simple Code}
	Nachfolgend ist der Code für den Test der Ordnungsgemäßen Funktion der Lichtschranken aufgeführt.
	}
	\begin{lstlisting}[language=C++, , xleftmargin=0pt, xrightmargin=0pt]
	#include <Bounce2.h>	
	// Pins definieren
	#define LICH_SCH_PIN 2 
	//GPIO Pin fuer Signal LS
	#define LED_PIN 13 
	// GPIO Pin fuer Versorgung LED, R!
		
	// Bounce-Objekt fuer Entprellung
	Bounce debouncer = Bounce();
	as Bounce-Objekt sowie den Serial-Monitor.

	void setup() {
	pinMode(LICH_SCH_PIN, INPUT_PULLUP); 
	// Konfigurieren als Input Pin, Pullup NC 
	pinMode(LED_PIN, OUTPUT); 
	//Konfigurieren der LED als Output 
			
	debouncer.attach(LICH_SCH_PIN); 
	//Zuweisung Debounce auf LS
	debouncer.interval(50); 
	//50 ms Entprellintervall 
			
	Serial.begin(9600); // Beginn der Kommunikation mit BAud Rate 9600 
	}
	void loop() { // Funktionen sind hier 
	debouncer.update();
			
	if (debouncer.read() == LOW) {     
	// Wenn der LS LOW - LED HIGH 
	digitalWrite(LED_PIN, HIGH);
	Serial.println("Objekt erkannt!"); 
	// Ausgabe im Command Window 
	} else {
	digitalWrite(LED_PIN, LOW); 
	// Normalerweise LED aus 
	}
	}
	\end{lstlisting}

\STANDARD{Test der Bibliothek}{
\subsection{Test all Functions}
Nachdem nun der Sensor auf seine grundlegenden Funktionen getestet wurde, ist es nun sinnvoll weitere Tests durchzuführen. Im Beispiel der Anwendung zum testen des Sensor wurde ebenfalls eine Bibliothek zum entprellen des Signals verwendet. Um Ihre ordnungsgemäße Einbindung und Funktion auf seiten der Hardware zu prüfen kann folgendes unternommen werden: 
\begin{enumerate}
	\item Erhöhen der Entprellzeit
	\item Aufspielen des Skriptes auf den Mikrokontroller 
	\item Bewegen eines Gegenstandes durch die Lichtschranke mit hoher Geschwindigkeit 
	\item Beobachten der Reaktion
\end{enumerate}
\textbf{Diesen Vorgang kann man nun mehrmals durchführen. Die Entprellzeit sollte hierbei bei jeder Iteration verringert werden. Die Geschwindigkeit mit der das Objekt durch die Lichtschranke geführt wird sollte hierbei immer konstant sein. Das Ganze solange wiederholen bis die Lichtschranke nicht mehr schaltet. Empfohlen werden Intervalle der Entprellzeit von ungefäh 50 ms.} 
Durch diesen Vorgang ist es möglich festzustellen, ob die eingebundene Bibliothek die Signale wirklich entprellt. Die gleiche Vorgehensweise mit einer Erhöhung der Entprellzeit kann ebenfalls diese Erkenntnis hervorbringen.}
